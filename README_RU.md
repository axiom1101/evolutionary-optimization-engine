**Evolutionary Optimization Engine (EOE)** — это предметно-независимый (domain-agnostic) AI-фреймворк, разработанный для решения NP-трудных задач оптимизации топологии, поиска скрытых синергий в сложных системах и алгоритмического арбитража с использованием мульти-агентной оркестрации LLM и генетических алгоритмов.

## 🎯 Концепция и Предметно-независимая Архитектура

Хотя ядро движка изначально прототипировалось для оценки сложных математических моделей с миллионами перестановок, его архитектура строго **универсальна (Domain-Agnostic)**. Система проецирует абстрактные узлы графа, модификаторы и фитнес-функции на реальные бизнес-вертикали.

| Абстрактный концепт | 📈 FinTech / Quant Trading (Текущая реализация) | ☁️ Cloud / MLOps | 🚚 Логистика и Цепочки поставок |
| :--- | :--- | :--- | :--- |
| **Граф топологии** | Граф корреляций и зависимостей активов | Топология микросервисов / сети | Логистическая и распределительная сеть |
| **Узлы / Компоненты** | Финансовые инструменты (Акции, Деривативы) | Вычислительные узлы (EC2, GPU, БД) | Транспортные хабы, Склады |
| **Модификаторы (Правила)** | Волатильность рынка, Налоги, Комиссии | Ограничения SLA, Сетевые задержки | Таможня, Погода, Стоимость топлива |
| **Фитнес-функция** | Ожидаемая доходность (Alpha) vs. Просадка | Пропускная способность vs. Отказоустойчивость | Объем поставок vs. Операционные расходы |
| **Поиск арбитража** | Поиск недооцененных комбинаций активов | Арбитраж цен на Spot-инстансы | Неэффективности в стоимости маршрутов |

## 🏗️ Архитектура Системы: LLM-Driven Multi-Agent Workflow

Главная ценность проекта — это надежная связка между сырыми данными, локальными/облачными LLM и специализированными промптами (Agentic Workflow).

### 1. ETL & Data Ingestion Layer (`/ingestion`)
*   **Legacy Data Parsing:** Извлечение данных из неструктурированных Lua/XML файлов с помощью регулярных выражений и лексического анализа (`slpp`) без риска выполнения произвольного кода (RCE).
*   **Data Normalization:** Скрипты глубокой валидации (`validate_lossless.py`), гарантирующие нулевую потерю данных (Zero Data Loss) при приведении к единому JSON-контракту.

### 2. LLM Integration & MLOps (`/agents/llm_client.py`)
*   **Local & Cloud Inference:** Поддержка гибридного запуска моделей. Интеграция с локальными моделями через **Ollama** (Qwen, Mistral) для приватных данных и с **Google Gemini API** для мультимодальных задач (Computer Vision).
*   **Prompt Engineering:** Использование жестких системных промптов (System Prompts) для принудительного возврата структурированных данных (JSON) и предотвращения галлюцинаций.

### 3. Multi-Agent Orchestration (`/agents`)
*   **Intent Router (NLP-маршрутизатор):** Анализ естественного языка пользователя для определения намерения (Intent) и извлечения сущностей (Entities) перед вызовом тяжелых моделей.
*   **Specialized Agents:** Разделение ролей (Стратег, Архитектор, Валидатор). Каждый агент имеет свой изолированный контекст и промпт.

### 4. Evolutionary Core (`/core/evolution_engine.py`)
*   **Генетические алгоритмы:** Реализует инициализацию популяции, кроссовер и направленные мутации для эволюции топологий системы.
*   **Элитизм и Wildcards:** Защищает наиболее эффективные конфигурации, сохраняя при этом генетическое разнообразие (10-15% wildcards) для выхода из локальных оптимумов.

### 5. In-Memory Data Grid & Evaluation (`/core`)
*   **Движок агрегации метрик:** Надежный класс `ModDB`, обрабатывающий сложную математику модификаторов (базовые, аддитивные, мультипликативные) для тысяч узлов.
*   **Мост глубокой симуляции:** Использует `lupa` (Lua-Python FFI) для связи Python-оркестратора с высокопроизводительным C/Lua движком симуляции (Цифровой Двойник).

### 6. Quantitative Arbitrage & RPA (`/automation`)
*   **Мониторинг сигналов в реальном времени:** Интеграция RPA и компьютерного зрения (`follower.py`) для автоматизированного сбора данных и выполнения действий в legacy-терминалах.

## 🛠️ Технологический стек
*   **Backend:** Python 3.11+
*   **Data Processing:** Regex, JSON Schema, XML ElementTree
*   **FFI / Simulation:** `lupa` (Lua in Python)
*   **AI / NLP:** `ollama` (Локальный запуск LLM), `google-genai` (Мультимодальное зрение)
*   **Automation:** `pyautogui`, `pygetwindow`
